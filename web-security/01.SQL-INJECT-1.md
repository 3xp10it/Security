防护

  SQL注入漏洞，是由于没有对数据语句做适当处理（以及）以及没有过滤用户数据而导致用户的输入成为SQL语句的一部分执行，从而实现控制数据库系统的目的。
  
  考虑下面一个SQL语句：
  
```java
 String sql = "select * from user where id = '" + id + "'";  
```

  id是用户传过来的参数，我们假设这里的id是String类型。那么正常情况下，假如ID为1，则最终的SQL语句为：

```sql
  select * from user where id = '1';
```
